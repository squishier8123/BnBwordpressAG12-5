# Project Rules & Agent Calibration Settings

## 1. High-Level Objective
- **Goal**: Audit live hunting directory site (https://beardsandbucks.com/) for launch readiness.
- **Scope**: Comprehensive inspection to identify broken features, performance issues, and mobile usability problems.
- **Constraint**: **INSPECTION ONLY**. DO NOT make any changes to the site.

## 2. Technology Stack
- **Platform**: WordPress + Elementor (page builder)
- **Payments**: Stripe
- **Hosting**: Hostinger
- **Tools**: Browser-based inspection agents.

## 3. Agent Personality & Workflow
- **Autonomy**: **FULLY AUTONOMOUS**. Do not ask for permission before proceeding with checks. Execute the full audit.
- **Verbosity**: **Just the facts**. Report working (✓) vs broken (✗). Include severity, location, and impact. No flowery language.
- **Speed**: Move fast.

## 4. Operational Rules
- **Mobile-First**: Primary focus is on mobile viewport (375px).
- **Aesthetics Check**: Verify "Rustic Midwestern hunting" aesthetic (earth tones: olive, brown, tan, sage).
- **Deliverable**: Structured report with Critical/Important/Minor issues.

## 5. Audit Phases
1.  **Foundation**: HTTPS, Console Errors, Load Time (<3s).
2.  **Core Pages**: Home, Directory, Vendor Detail, Used Gear, Forms.
3.  **Functionality**: Search, Filters, Navigation (Mobile).
4.  **Design**: Branding, Responsiveness.
5.  **Security**: HTTPS, Policies.
6.  **Performance**: 4G simulation, responsive breakpoints.
7.  **Cross-Browser**: Compatibility checks.
8.  **Summary**: Prioritized issue list.
