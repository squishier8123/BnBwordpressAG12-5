**Date:** 2025-12-05
**Based on:** Antigravity Audit Report
**Status:** 9 actionable items prioritized for launch readiness

## Executive Summary

Antigravity audit identified **4 critical issues** blocking site functionality:
1. Directory listings not displaying
2. Search widget broken
3. Vendor detail pages inaccessible
4. Missing legal pages (Privacy/Terms)

This fix plan addresses all issues in priority order. **Estimated time: 3-4 hours total.**

---

## Priority 1: CRITICAL BLOCKERS (Must fix before site can function)

### 1. Fix Directory Listings Display
**Severity:** CRITICAL üî¥
**Impact:** Users cannot browse vendors at all
**What's broken:** `/directory` and `/find-vendors` pages load empty
**Root cause:** Elementor shortcodes or Listeo plugin not rendering vendor data

**Steps to fix:**
1. Go to your WordPress admin
2. Check the Directory/Find Vendors page in Elementor
3. Look for shortcodes like `[listeo_vendors]` or similar
4. Verify Listeo plugin is active and configured
5. Check Listeo plugin settings (vendor query, display options)
6. Test the shortcode - ensure vendor data exists in the database
7. If vendors don't exist, create test vendors first
8. Rebuild the directory page in Elementor if needed

**Success criteria:** Vendors appear in listing grid on both mobile (375px) and desktop

---

### 2. Create Privacy Policy Page
**Severity:** CRITICAL üî¥
**Impact:** Legal/compliance violation - blocks Stripe integration
**What's missing:** No Privacy Policy page or footer link

**Steps to fix:**
1. Go to WordPress admin ‚Üí Pages ‚Üí Add New
2. Create page titled "Privacy Policy"
3. Add your privacy policy content (standard template):
   - What data you collect
   - How you use it
   - GDPR/privacy compliance statements
   - Contact info for privacy inquiries
4. Publish the page
5. Note the page URL (e.g., `/privacy-policy/`)
6. Go to Appearance ‚Üí Menus
7. Create or edit "Footer" menu
8. Add Privacy Policy page to footer menu
9. Assign menu to "Footer" menu location

**Success criteria:** Privacy Policy link appears in footer on all pages

---

### 3. Create Terms of Service Page
**Severity:** CRITICAL üî¥
**Impact:** Legal/compliance violation - blocks Stripe integration
**What's missing:** No Terms & Conditions page or footer link

**Steps to fix:**
1. Go to WordPress admin ‚Üí Pages ‚Üí Add New
2. Create page titled "Terms of Service" (or "Terms & Conditions")
3. Add your terms content:
   - User account terms
   - Payment/refund policies
   - Vendor listing policies
   - Liability disclaimers
   - Dispute resolution
4. Publish the page
5. Go to Appearance ‚Üí Menus
6. Edit "Footer" menu
7. Add Terms of Service page to footer menu
8. Save menu

**Success criteria:** Terms link appears in footer on all pages alongside Privacy Policy

---

### 4. Fix Search Widget
**Severity:** CRITICAL üî¥
**Impact:** Users cannot search for vendors or gear
**What's broken:** Search bar appears but is non-functional/non-editable

**Steps to fix:**
1. Go to WordPress admin
2. Navigate to the homepage in Elementor editor
3. Find the search widget/form
4. Check if it's using Elementor's built-in search or a plugin's shortcode
5. Verify search input is connected to a search results page
6. Test: Try searching for a vendor or gear item
7. Check browser console for JavaScript errors (F12 ‚Üí Console tab)
8. If using custom search plugin, verify it's configured and has search results page set
9. If using Elementor search, ensure it has a target search page

**Success criteria:** Users can type in search bar and get results

## Priority 2: IMPORTANT ISSUES (Should fix before launch)

### 5. Fix Font File 404 Errors
**Severity:** IMPORTANT ‚ö†Ô∏è
**Impact:** Typography rendering issues, slower load times
**What's broken:** Raleway font woff2 files returning 404 errors

**Steps to fix:**
1. Go to WordPress admin ‚Üí Appearance ‚Üí Customize
2. Look for Typography or Font settings
3. Check if Raleway font is properly loaded from CDN (Google Fonts)
4. If using custom fonts folder, verify files exist in `/fonts/` directory
5. Check browser console (F12) - look for which font files are 404ing
6. Either:
   - Use Google Fonts CDN (recommended): Add to theme header
   - Or: Upload missing font files to server and update CSS paths
7. Clear browser cache and reload site

**Success criteria:** No 404 errors in console for font files

### 6. Consolidate Duplicate Pages
**Severity:** IMPORTANT ‚ö†Ô∏è
**Impact:** Confusion about which page to use, duplicate content
**What's broken:** Both `/directory` and `/find-vendors` attempt same functionality

**Steps to fix:**
